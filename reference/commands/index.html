Command Reference

The SQI engine recognizes 68 commands in its dispatch table. Every command follows the pattern @COMMAND(args) and is resolved by replacing the entire expression with its return value. Commands that exist for side effects return an empty string.

Parameter Store (WIZ*)

The parameter store is an in-memory key-value map (501 slots). These commands read, write, and query it.

@WIZSET(key=value)

Always sets key to value, overwriting any existing value. Returns empty string.

WIZSET then WIZGET
Ctrl+Enter to run
Parameter Store

@WIZGET(key)

Returns the value for key, or empty string if not found.

WIZGET a missing key (returns empty)
Ctrl+Enter to run
Parameter Store

@WIZINIT(key=value)

Sets key to value only if key does not already exist. Use this to set defaults without overwriting caller-provided values.

WIZINIT does not overwrite existing keys
Ctrl+Enter to run
Parameter Store

@WIZRPL(key=value)

Despite the name “replace”, this appends value to any existing value. If key is missing, it creates a new entry.

WIZRPL appends to existing value
Ctrl+Enter to run
Parameter Store

@WIZAPPEND(key=value)

Same behavior as @WIZRPL — appends value to existing key or creates a new entry.

@WIZCLR(key)

Removes the key-value pair entirely from the store. Returns empty string.

WIZCLR removes a key
Ctrl+Enter to run
Parameter Store

@WIZCLRALL(prefix)

Deletes all keys whose name starts with the given prefix.

@WIZISBLANK(key)

Returns "1" if key is missing or has an empty value. Returns "0" if key exists with a non-empty value.

WIZISBLANK on a non-empty key
Ctrl+Enter to run
Parameter Store

@WIZEXIST(key)

Returns "1" if key exists in the store (regardless of value), "0" if not.

WIZEXIST returns 1 even for empty values
Ctrl+Enter to run
Parameter Store

@WIZEXISTALL(prefix)

Returns "1" if any key starts with the given prefix, "0" if none found.

WIZEXISTALL prefix scan
Ctrl+Enter to run
Parameter Store

@WIZRESET

Preprocessor variable (not a command with parentheses). Clears all keys from the parameter store.

Format and Control Flow (@FMT)

The @FMT command is a multi-purpose dispatcher for formatting, comparison, string operations, encoding, and check digit calculations. It has 33 sub-commands documented in detail on the @FMT Sub-commands page.

Quick Summary

Sub-commandPurposeExample

CHRCharacter by ASCII code@FMT(CHR,65) — “A”
LENString length@FMT(LEN,hello) — “5”
POSFind substring@FMT(POS,lo,hello) — “4”
Sn:mSubstring@FMT(S1:3,hello) — “hel”
!c:wPad/zero-fill@FMT(!0:4,42) — “0042”
#:dNumber format@FMT(#:2,3.14159) — “3.14”
UPPER/LOWERCase conversion@FMT(UPPER,hello) — “HELLO”
TRIMTrim whitespace@FMT(TRIM, hello )
2HTML/HTML2HTML encode/decode@FMT(2HTML,<b>)
2URL/URL2URL encode/decode@FMT(2URL,a b)

Database Access

@DBHOT(mode,…)

The primary database and system state access command with 82 dispatch modes. Reads from datasets, INI files, hot memory, device info, and more. See the full @dbHot reference.

Note

Database commands return empty strings or mock data in the browser REPL. They require a live POS database connection in production.

@DBSELECT(sql_query)

Execute a SQL query, return a single scalar value.

@DBEXEC(sql_statement)

Execute SQL directly. Supports a RESULT prefix to capture results. Returns empty string.

@DBTABLE(params)

Database table operation.

@DBAUTO(params)

Generate auto-number value for a database field.

@DBAUTOHDR(params)

Generate auto-number for a header record.

@DBVIEW(name,sql_query)

Create a named in-memory view from a query result.

@DBROW(viewname,column,format)

Read a column from a named view created by @DBVIEW.

@DBFIND(params)

Database find/search operation.

@DBGEN(field,increment)

Generate next sequence number for a field.

@DBFLD(table,prefix,exclude_fields)

Generate a field list for a table, optionally excluding specified fields.

@DBDCT(field_list)

Get data dictionary / column definitions for a list of fields. Used for CREATE TABLE statements.

@DBCOL(params)

Database column information.

@DBPLAN(params)

Database plan operation.

@DBSND

Recognized but does nothing (no-op).

@DBSET

Recognized but does nothing (no-op).

Execution (@EXEC)

@EXEC(params)

Execute POS functions, display messages, run scripts, and control peripherals. The first parameter determines the sub-command type. See the full @EXEC reference.

ParameterDescription
FCT=codeExecute POS function by code number
MSG=textDisplay message (prefix controls type)
SQI=nameExecute another SQI script
XCH=commandExchange/communication command
CGI=nameExecute CGI/HTML template
HTT=nameHTML template in form window
INI=sectionLoad INI configuration section
TASK=nameExecute named task
BYE=modeExit mode (REC, INV, EXIT)

Note

All @EXEC operations are no-ops in the browser REPL. They require the full POS runtime.

Import

@IMPORT_SIL(filename)

Include another .sqi file inline via text substitution. The included file’s content replaces the @IMPORT_SIL(...) call, then processing continues on the expanded text.

Report Accumulators

@RPTCLR(varname)

Clear/reset a named report accumulator variable. Supports comma-separated lists for clearing multiple accumulators at once.

@RPTTTL(varname,format)

Get the accumulated value of a report variable. Supports arithmetic expressions and optional format specifiers.

@RPT_DETAIL(params)

Report detail. Internally prepends “DETAIL” to arguments, then processes.

@RPTSET(varname,value)

Set a report variable value directly. Supports BINARY flag for binary data.

Utility Commands

@TOOLS(subcommand,args…)

Multi-purpose utility dispatcher with sub-commands for parameter parsing, XML/JSON, receipts, encoding, dialogs, validation, WIC limits, smart cards, and more. Key sub-commands:

Sub-commandPurpose
PARAM,,n,stringExtract parameter N from delimited string
ENCODE / DECODEString encoding/decoding
ENCODE64 / DECODE64Base64 encode/decode
MESSAGEDLG,'text',,buttonsShow message dialog, return button result
CHECKPRMPermission check
RAISEERROR,typeRaise typed error (USERLEVEL, NOTINTABLE, etc.)
SQLLIKE,pattern,valueSQL LIKE pattern matching
CALCDATE,exprDate calculation
HASHDATA / HMAC_SHA256Cryptographic hash operations
RCT_LINE / RCT_BARCODEReceipt line / barcode printing

@READKEY(position)

Check if a license key permission bit is enabled. Returns "1" if the feature at the given position is licensed.

@POST(FCT=code,CN=connection)

Post a command asynchronously to another terminal/connection via circular buffer and Windows IPC.

@DATE(params)

Date manipulation function.

@NULL(params)

Null/empty value handler. Used for date normalization (returns empty if value contains “1899”, the null date epoch).

@SECURE(params)

Security and permission check.

Batch Operations

@UPDATE(params)

Update existing database record.

@UPDATE_VIEW(params)

Update via view. Equivalent behavior to @UPDATE.

@UPDATE_BATCH(params)

Batch database update with structured parameters:

ParamDescription
JOB=ADDOperation type (add/update records)
TAR=tableTarget table
KEY=conditionMatching key condition
SRC=SELECT...Source query

@UPDATE_STACK(params)

Update stack operation.

@UPDATE_REPORT

Recognized but has no effect (no-op).

@CREATE(params)

Create new database record.

@CREATE_INDEX(params)

Create database index.

@INSERT_HOR(params)

Insert horizontal (cross-table insert).

@ACTIVATE_HEADER(params)

Activate a header record.

Wizard UI Commands

These commands configure the wizard form UI that appears in the POS application. They all behave similarly and manipulate form layout.

@WIZDETAIL

Wizard detail section marker.

@WIZDATE

Single date picker control.

@WIZDATES

Dual date picker (start/end date range).

@WIZTARGET

Store/target destination selector.

@WIZTLZ

Payment type / totalizer selector.

@WIZFIL

Filter/edit field configuration.

@WIZLBL

Label display (read-only text) or label file selector (.LBZ).

@WIZEDIT

Edit field configuration.

@WIZMENU(key=title,option1=value1,option2=value2,…)

Create a selection menu. The selected value is stored under the specified key.

Messaging Commands

@WINMSG(params)

Display a window message.

@WINMAIL(params)

Send a window mail message.

@WINWAIT(params)

Wait for a window event or response.

@MSGTABLE(params)

Message table operation.

@MSGROW(params)

Read a row from a message table.

@MSGFILE(params)

Message file operation. Supports BINARY flag for binary data.

@MSGSUM(params)

Message summary/aggregation.

@MSGDEL(params)

Delete message.

@MSGLOG(params)

Write to message log.

Macro Commands

@MACSHORT(params)

Re-invoke the preprocessor for nested macro expansion. Useful when a runtime-constructed string contains preprocessor variables that need resolution.

@MACSIL(sql_statement)

Execute SQL with macro/SIL processing — expands embedded macros before execution.

Text Processing

@REPLACE_START(pattern)

Begin a text replacement block. Text between this and @REPLACE_STOP has pattern-based replacements applied.

@REPLACE_STOP

End the text replacement block.

MIME Operations

@MIMESET(params)

Set MIME parameters for file operations:

ParamDescription
TAR_PATH=Target path
TAR_OPT=Target options
TAR_FILE=Target filename

Script Control

@ATTRIB(params)

Set script attributes controlling escape handling and EOF behavior.

@IF

Conditional — recognized but has no effect (vestigial/no-op). Use @FMT(CMP,...) for conditionals.

Database Template

@DBRPT(params)

Database report section. Used in HTML templates to define report regions.

Data Mapping

@FIELDS_FROM_QUERY

Extract field definitions from a query result.

@MAP_FROM_QUERY

Create field mapping from a query.

@MAP_FROM_VIEW

Create field mapping from a named view.

@MAP_BACKUP(params)

Backup the current field mapping.
