@FMT Sub-commands

The @FMT() command is the Swiss army knife of SQI. Its first argument selects from 33 sub-commands covering conditionals, string manipulation, number formatting, encoding, and check digit calculation.

Syntax: @FMT(sub_command, arg1 [, arg2 [, arg3]])

Conditional / Control Flow

CMP — Comparison

Syntax: @FMT(CMP, condition, true_branch, false_branch)

The primary conditional construct. Evaluates the condition and returns the matching branch.

Operators:

OperatorMeaning
=Equal
<>Not equal
>Greater than
<Less than
>=Greater or equal
<=Less or equal

Comparison is case-insensitive (uses case-insensitive string comparison).

Basic equality check

Caution

Inside CMP branches, the @ character cannot be used directly because the engine would resolve it before CMP chooses a branch. Use the branch delimiter character (typically c-cedilla U+00E7) as a placeholder for @. After branch selection, the delimiter is replaced with @ and the revealed commands are processed.

CHR — Character Code

Syntax: @FMT(CHR, ascii_code)

Returns the character for the given ASCII code. Two codes have special meaning for script flow control:

CodeCharacterEffect
26SUB (^Z)Exit script with success — continue POS flow
27ESCAbort script with error — cancel operation
64@Literal @ character (avoids preprocessor)
OtherliteralThe actual character
CHR(65) = 'A'

MARGIN — Margin Calculation

Syntax: @FMT(MARGIN, cost, retail, margin_pct)

Computes: ((value - cost) * 100 / value) * multiplier

RND — Random Number

Syntax: @FMT(RND, format)

Generates random formatted numbers. Supports barcode-style random generation.

Null / Value Checking

NUS — Number String

Syntax: @FMT(NUS, value) — returns numeric string, or empty if value is “n/a”

NUL — Null Check

Syntax: @FMT(NUL, value) — returns empty if zero or “n/a”, otherwise the value

NUD — Null Date Check

Syntax: @FMT(NUD, date_string) — returns empty if value contains “1899” (null date epoch), otherwise the value

Check Digit Algorithms

MOD10 — Luhn Algorithm

Syntax: @FMT(MOD10, number_string) — returns check digit

Standard Luhn algorithm used for credit card validation and barcode check digits.

Luhn check digit
Ctrl+Enter to run
Parameter Store

MOD10P — Paired Modulo 10

Syntax: @FMT(MOD10P, number_string) — variant check digit calculation

UPCA — UPC-A Check Digit

Syntax: @FMT(UPCA, barcode) — returns UPC-A formatted with check digit

UPC / UPC0 — Barcode Formatting

Sub-commandDescription
UPCAuto-detect format (param -1)
UPC0Explicit format (param 0)

GPC — General Product Code

Syntax: @FMT(GPC, code) — formatted product code with “00” prefix handling

HUDON — Hudson’s Bay Check Digit

Syntax: @FMT(HUDON, number) — Hudson’s Bay Company format. Pads to 5 digits, applies weighted calculation.

HENING — Henning Check Digit

Syntax: @FMT(HENING, number) — Henning format. Pads odd-length numbers, iterates digit pairs.

Version Parsing

VER — Version String

Syntax: @FMT(VER, version_string)

Parses version strings by stripping “VERSION”/“VER”/“V” prefixes, splitting by , then ., and extracting M (major), R (revision), B (build).

Flag / Boolean

FLG — Flag Check

Syntax: @FMT(FLG, value)

If value equals "1" and no third parameter: returns "checked" (for HTML checkbox attributes). Otherwise returns empty string.

SQL Compatibility

MSSQL6 — MSSQL Format Switch

Syntax: @FMT(MSSQL6, value, true_result, false_result)

Returns true_result or false_result based on a system MSSQL compatibility config byte.

ISSQL — SQL Check

Syntax: @FMT(ISSQL, value) — passes through value with special character replacement.
