@EXEC Operations

The @EXEC() command dispatches POS operations: running functions, displaying messages, executing scripts, controlling peripherals, and managing transactions. The @TOOLS() command provides utility functions for parameter parsing, encoding, receipts, and validation.

Note

All @EXEC and @TOOLS operations are no-ops in the browser REPL. They require the full POS runtime environment. This page documents their syntax and production behavior.

@EXEC — Command Execution

FCT — Function Code Dispatch

Syntax: @EXEC(FCT=code)

Executes a POS function by numeric code. The system checks for registered SQI hook scripts that run on specific function+event combinations.

CodeDescription
30Void/cancel transaction
104Function key
119WIC tender
200-270Payment functions (tender types)
307Process entry (item lookup)
330Header operation
350Display during processing
380Open transaction by number
385Transaction close
710Third-party payment initiation
720Subtotal
725Print sub-total
727Payment complete
730Print receipt
740Payment finalize

MSG — Message Display

Syntax: @EXEC(MSG=text)

The first character of the message text controls the display behavior:

PrefixBehavior
!Error — blocking dialog, requires acknowledgment
~Timed info — non-blocking notification, records timestamp
?Query — interactive dialog, reads MSG_TYPE/MSG_TAG params
#Log — writes to script log (debug only, no display)
+Extended — formatted popup with MSG_URL, MSG_LVL, MSG_ID
^Sound — plays WAV file via Windows PlaySoundA
*External display — sends to external display device
_Clear — clears message area
(space)Clear — clears message area
(plain)Standard — displays text in message area
(empty)Clear — clears message display
text?Trailing query — text ending with ? is treated as query

Syntax: @EXEC(SQI=filename)

Execute another SQI script file.

@POST — Post to Other Terminal

Syntax: @POST(FCT=code, CN=connection)

Posts a command asynchronously to another terminal/connection. Uses a circular buffer (200 slots) and a Windows message for IPC.

Behavior:

  1. If parameter starts with "POST=1,": strips prefix, uses normal priority
  2. Otherwise: uses priority posting
  3. Acquires mutex with 20-second timeout
  4. Stores command in circular buffer
  5. Posts Windows message to trigger processing
  6. Releases mutex

@READKEY — License Key Check

Syntax: @READKEY(position) — returns "1" if feature at position is licensed

Reads a specific bit/position from the license key bitmap.

Report Accumulator Commands

@RPTCLR(varname)

Clear/reset a named report accumulator. Supports comma-separated list for clearing multiple accumulators.

@RPTTTL(expression,format)

Read and format a report accumulator value. Supports arithmetic expressions.

@RPTSET(varname,value)

Set a report variable value. Supports BINARY flag for binary data.
