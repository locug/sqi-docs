@EXEC Operations

The @EXEC() command dispatches POS operations: running functions, displaying messages, executing scripts, controlling peripherals, and managing transactions. The @TOOLS() command provides utility functions for parameter parsing, encoding, receipts, and validation.

Note

All @EXEC and @TOOLS operations are no-ops in the browser REPL. They require the full POS runtime environment. This page documents their syntax and production behavior.

@EXEC — Command Execution

FCT — Function Code Dispatch

Syntax: @EXEC(FCT=code)

Executes a POS function by numeric code. The system checks for registered SQI hook scripts that run on specific function+event combinations.

CodeDescription
30Void/cancel transaction
104Function key
119WIC tender
200-270Payment functions (tender types)
307Process entry (item lookup)
330Header operation
350Display during processing
380Open transaction by number
385Transaction close
710Third-party payment initiation
720Subtotal
725Print sub-total
727Payment complete
730Print receipt
740Payment finalize

MSG — Message Display

Syntax: @EXEC(MSG=text)

The first character of the message text controls the display behavior:

PrefixBehavior
!Error — blocking dialog, requires acknowledgment
~Timed info — non-blocking notification, records timestamp
?Query — interactive dialog, reads MSG_TYPE/MSG_TAG params
#Log — writes to script log (debug only, no display)
+Extended — formatted popup with MSG_URL, MSG_LVL, MSG_ID
^Sound — plays WAV file via Windows PlaySoundA
*External display — sends to external display device
_Clear — clears message area
(space)Clear — clears message area
(plain)Standard — displays text in message area
(empty)Clear — clears message display
text?Trailing query — text ending with ? is treated as query

SQI — Execute Script

Syntax: @EXEC(SQI=filename)

Execute another SQI script file. If no file extension is provided, .sqi is appended automatically. All current params from the calling script are available in the called script.

SQL / USP / SQR / SQM / SQT — Related Script Types

The system supports multiple script formats beyond SQI, each with its own file extension and handler:

Sub-commandExtensionDescription
SQL=name.sqlSQL script (pure SQL with macro extensions). Path macros like @RUN and @OFFICE are resolved.
USP=name.uspUser Stored Procedure script. Looks up in office path if no explicit path. Reads OUTPUT param for result target.
SQR=name.sqrSQR report script
SQM=name.sqmSQM macro script. Path macros resolved.
SQT=name.sqtSQT template script. Path macros resolved.

Syntax: @EXEC(XCH=command)

Exchange and communication operations. The command string selects the operation type. Dispatched in two levels: named device/session commands first, then file operations as fallback.

RECVFTP receive/download

Additional file operation parameters:

ParameterDescription

EXE / RUN — External Program Execution

Syntax: @EXEC(EXE='command') or @EXEC(RUN='command')

Launches an external program via CreateProcessA. The system sets STOREMAN and OFFICE environment variables before launching.

Sub-commandWait BehaviorUse Case
EXE='cmd'Blocks until process exitsBatch scripts, backups, tools that must complete
RUN='cmd'Returns once app is idleLaunch apps that stay open, async operations

CLIEXE / CLIRUN — SmsCli.exe Execution

Syntax: @EXEC(CLIEXE='args') or @EXEC(CLIRUN='args')

Executes commands through SmsCli.exe (the CLI companion tool). CLIEXE waits for exit; CLIRUN returns immediately.

LNK — Shell Link

Syntax: @EXEC(LNK=target)

Executes a Windows shell link (shortcut/.lnk file).

HOST — Remote Host Routing

Syntax: @EXEC(HOST=..., params...)

Routes the EXEC command to a remote host server. Checks SMS_SERVERNAME and CRIND_SERVERNAME params:

  1. If the server name matches the local server, dispatches locally
  2. Otherwise builds an HTTP request with LOGIN=, PASSWORD=, and DN=XML and sends to the remote host
  3. Parses XML response for PARAM/MESSAGE — if present, shows as error dialog
  4. Otherwise creates a new form and executes the command locally with the remote connection params

ENTRY — Set Entry Buffer

Syntax: @EXEC(ENTRY=value)

Sets the entry buffer (the “scan line” input field) to a specific value.

DRL — Cash Drawer

Syntax: @EXEC(DRL=...)

Opens the cash drawer.

KEY — Key Injection

Syntax: @EXEC(KEY=keyname)

Injects a key press into a specific form. Reads the FRAME param to identify the target form.

INI — Load INI Configuration

Syntax: @EXEC(INI=section)

Reads an INI file section and applies its settings. The parameter specifies the file and section in filename[section] format. Key=value pairs from the section are processed as configuration directives.

HTT — HTML Template in Form Window

Syntax: @EXEC(HTT=name)

Renders an HTML template file in a form window. Supports PUBLIC\ prefix for shared templates. Path macros (@RUN, @OFFICE) are resolved. If no explicit path, prepends the profile’s Cgi\ directory.

Reads OUTPUT param to determine output target. If not set, generates an HTML session file. The template can contain SQI macros, database queries, and dynamic content.

HTI — HTML Template Inner Frame

Syntax: @EXEC(HTI=name)

Like HTT but renders in the inner body area of an existing form. Reads HTI_ID param for targeting specific inner frames.

HTI=BODYBLANK blanks the body area of the specified form.

CGI — CGI Template Execution

Syntax: @EXEC(CGI=name)

Similar to HTT with additional logic for device-direct (inner-frame) rendering. Defaults OUTPUT to STRING if not set. Supports device-direct mode where output goes directly to a connected device.

CGS — CGI Script (Server-Side)

Syntax: @EXEC(CGS=name)

Server-side CGI script execution. Like CGI but without the inner-frame device-direct logic. Sets default OUTPUT to STRING.

CGP — CGI Print

Syntax: @EXEC(CGP=name)

CGI print operation — sends CGI output directly to the print handler.

HTA — HTML Application

Syntax: @EXEC(HTA=name)

Loads an HTML Application template. Uses Hta\ path prefix instead of Cgi\. Otherwise similar to HTT processing.

XLS — Excel Export

Syntax: @EXEC(XLS=name)

Excel/spreadsheet export operation.

XAML / XAMLI — XAML Template Rendering

Syntax: @EXEC(XAML=name) or @EXEC(XAMLI=name)

XAML template rendering for WPF-based UI panels:

  • XAML=name renders XAML in a form window. Handles PUBLIC\ prefix. Checks for .xaml extension.
  • XAMLI=name renders XAML in an inner frame. XAMLI=BODYBLANK blanks the body.

FRM — Form Management

Syntax: @EXEC(FRM=subcommand)

Manages form creation, hooks, and mode switching.

Sub-commandDescription
CREATECreate a new form. Reads FORM param for name. Fires EVT=LOGIN on connection 0, otherwise fires specified EVENT.
XCREATELike CREATE but for XAML forms. Reads Frame param.

CLOSEALLClose all forms
CAMERAOpen camera view
CAMERA_OFFClose camera view

HOOKAC1Trigger account hook

HOOKINVTrigger inventory hook
RESUMELASTResume last suspended transaction

If no sub-command matches, iterates all forms to find one matching the name and shows it.

SSM — Session/Mode Switch

Syntax: @EXEC(SSM=mask)

Switches the POS to a different operating mode (selling, receiving, inventory). The mask is a multi-character mode string: the first 4 characters identify the mode, characters 5+ contain sub-mode information.

The system validates that no transaction is in progress (shows error if completing) and that training mode is not active.

FCT_ONLY — Function Code Without Hooks

Syntax: @EXEC(FCT_ONLY=code)

Like FCT but skips hook script dispatch. Useful when you need to execute a function code without triggering any registered SQI hook scripts.

BYE — Exit / Close Mode

Syntax: @EXEC(BYE=mode)

Exits a POS mode, closing connections and cleaning up transactions.

ModeDescription
RECClose all receiving-mode transactions
INVClose all inventory-mode transactions
EXITFull exit — broadcasts BYE=EXIT to other terminals
CLOSEClose a specific form (reads FORM param)
PARAMClear all parameters

HLP — Help Display

Syntax: @EXEC(HLP=code)

Displays a help page by numeric code.

@EXEC() — Empty Call

When called with no arguments, executes using current parameter store values (MSG or other pre-set params).

@TOOLS — Utility Commands

Syntax: @TOOLS(sub_command, param2, param3, param4, param5, ...)

The largest single dispatcher in the scripting engine with 132 sub-commands. Parameters are positional (1-based), with the first parameter selecting the sub-command and subsequent parameters passed to the handler. The return value becomes the @TOOLS() expression result.

If no sub-command matches, the raw expression text is returned unchanged — this makes typos silent rather than producing errors.

Tip

Two special sub-commands exist before the main dispatch chain: one delegates directly to @EXEC (letting @TOOLS proxy an @EXEC call while saving/restoring OUTPUT and STYLE params), and one handles an ENTRY key from the wizard params for connection routing.

Parameter Parsing

Commands for extracting, converting, and managing wizard parameters.

PARAM — Extract Delimited Field

Syntax: @TOOLS(PARAM, delimiter, index, string)

Extracts the Nth element (1-based) from a delimited string. If the delimiter is empty or absent, defaults to comma.

Extract 2nd underscore-delimited field
Ctrl+Enter to run
Parameter Store
Extract major version number
Ctrl+Enter to run
Parameter Store

PARAMTOLINE — Serialize All Params

Syntax: @TOOLS(PARAMTOLINE)

Converts all current wizard parameters into a single-line string representation. Serializes the entire parameter dictionary. Useful for capturing state before an operation that might modify params.

LOADPARAMTOLINE — Load INI Section as Param Line

Syntax: @TOOLS(LOADPARAMTOLINE, ini_section)

Loads an INI file section and converts it to a single-line param format. The argument specifies Filename.ini[SECTION]. Often piped directly into @EXEC():

PARAMTONODES — Params to XML Nodes

Syntax: @TOOLS(PARAMTONODES)

Converts wizard params to an XML node-tree structure.

PARAMTOINPUT — Params to Input Format

Syntax: @TOOLS(PARAMTOINPUT)

Converts wizard params to input-format (form field assignment format for HTML/CGI forms).

PARAMTOCOMA — Params to CSV

Syntax: @TOOLS(PARAMTOCOMA)

Converts wizard params to comma-separated value format.

PARAMTOURL — Params to URL Encoding

Syntax: @TOOLS(PARAMTOURL)

Converts wizard params to URL-encoded query string format (key=value&key2=value2).

COMATOURL — CSV to URL Encoding

Syntax: @TOOLS(COMATOURL)

Converts a CSV-format string to URL-encoded format.

COMATOINPUT — CSV to Input Format

Syntax: @TOOLS(COMATOINPUT)

Converts a CSV string to input format.

PARAMINBRAKET — Extract Bracketed Value

Syntax: @TOOLS(PARAMINBRAKET, string)

Extracts a parameter value that appears inside brackets. Used for INI-style [SECTION] extraction from path strings.

PARAMCOPYVALUE — Copy Parameter

Syntax: @TOOLS(PARAMCOPYVALUE, source_param, dest_param)

Copies a wizard parameter value from one key to another.

PARAMPURGE — Remove Parameter

Syntax: @TOOLS(PARAMPURGE, param_name)

Removes a wizard parameter entirely from the parameter store.

PARAMSECURE — Mask Sensitive Parameters

Syntax: @TOOLS(PARAMSECURE, param_list)

Marks wizard parameters as “secure” so their values are masked in logs and displays. Accepts a comma-separated list of parameter names.

XML / JSON

Commands for generating SQL-safe XML values and extracting data from XML/JSON structures.

XMLSQL — SQL-Safe XML Value Formatting

Syntax: @TOOLS(XMLSQL, field_name, value)

Generates SQL-safe formatting for XML query values. Looks up the field type from the field registry and applies type-specific formatting:

Field TypeBehavior
D (Date)Converts to date format, then wraps for SQL
I (Integer)Passes through as-is (no quoting)
N / M (Numeric)Passes through as-is (no quoting)
String (default)Wraps in SQL quotes with proper escaping
ISNULLReturns empty string

Special handling exists for field index 1 (primary key).

REPORT_JSON — Report to JSON

Syntax: @TOOLS(REPORT_JSON, ...)

Converts report data to JSON format.

XML_VALUE — Extract XML Value

Syntax: @TOOLS(XML_VALUE, xml_string)

Extracts a value from an XML fragment.

XML_NODE — Extract XML Node

Syntax: @TOOLS(XML_NODE, xml_string)

Extracts a node from an XML fragment.

Receipt / Print

Commands for formatting and printing receipt content.

RCT_LINE — Format Receipt Line

Syntax: @TOOLS(RCT_LINE, text, alignment_code, label)

Formats a receipt line with alignment and padding. Reads the RCT_WIDTH wizard param (defaults to 40 columns). Calculates padding based on the width minus the label length.

  • Parameter 2: text content
  • Parameter 3: alignment/formatting code (e.g., %-40s)
  • Parameter 4: label/prefix text

RCT_REVERSE — Reverse String

Syntax: @TOOLS(RCT_REVERSE, text)

Reverses a string character-by-character. Used for right-to-left receipt formatting or barcode reversal.

RCT_QTY — Format Quantity

Syntax: @TOOLS(RCT_QTY, value)

Formats a quantity value for receipt printing.

RCT_AMT — Format Amount

Syntax: @TOOLS(RCT_AMT, value)

Formats an amount value for receipt display with currency formatting.

RCT_FLAG — Receipt Flag

Syntax: @TOOLS(RCT_FLAG, ...)

Sets or gets a receipt flag value that controls formatting behavior.

RCT_BMP / RCT_BMPS — Receipt Bitmap

Syntax: @TOOLS(RCT_BMP, bitmap_data) or @TOOLS(RCT_BMPS, bitmap_data)

Embeds a bitmap image into a receipt. RCT_BMPS adds scaling support for different printer resolutions.

RCT_RASTER — Raster Image

Syntax: @TOOLS(RCT_RASTER, image_data)

Converts image data to raster format for receipt printers. Processes the image row-by-row for printer-compatible output.

RCT_BARCODE — Receipt Barcode

Syntax: @TOOLS(RCT_BARCODE, barcode_data)

Generates a barcode image for receipt printing. Handles barcode symbology selection and printable output generation.

RCT_HTML — Receipt HTML

Syntax: @TOOLS(RCT_HTML, html_content)

Embeds HTML content into a receipt (for printers that support HTML rendering).

RCT_PUMP — Gas Pump Receipt Data

Syntax: @TOOLS(RCT_PUMP, ...)

Formats gas pump transaction data for receipt output. Multi-parameter handler for fuel sale details.

RCT_CARD — Card Payment Receipt Data

Syntax: @TOOLS(RCT_CARD, ...)

Formats card payment data for receipt output (masked card number, authorization code, etc.).

RCT_EFT — EFT Receipt Data

Syntax: @TOOLS(RCT_EFT, ...)

Formats Electronic Funds Transfer data for receipt output.

SLP_LINESPACING — Slip Line Spacing

Syntax: @TOOLS(SLP_LINESPACING, ...)

Sets slip printer line spacing for multi-part forms.

CARDBIN_FUNCTIONS — BIN Lookup

Syntax: @TOOLS(CARDBIN_FUNCTIONS, ...)

Card BIN (Bank Identification Number) lookup functions for identifying card issuer from the first 6-8 digits.

PRINT — Direct Print

Syntax: @TOOLS(PRINT, ...)

Sends data directly to the printer.

LABELPRINT — Print Label

Syntax: @TOOLS(LABELPRINT, ...)

Prints a label (shelf label, price label, etc.).

LABELPARAM — Set Label Parameters

Syntax: @TOOLS(LABELPARAM, format_path)

Sets label printing parameters by loading a label format definition file.

BIZERBA_LABEL — Scale Label

Syntax: @TOOLS(BIZERBA_LABEL, ...)

Generates a label for Bizerba commercial scales (meat/deli department scales with built-in label printers).

Encoding / Security

Commands for encryption, hashing, Base64, tokens, and certificate operations. The system supports multiple encoding modes internally (mode 1 = basic, mode 3 = stronger, mode 5 = Base64).

ENCODE — Encode String

Syntax: @TOOLS(ENCODE, data) or @TOOLS(ENCODE, data, wizard_key)

Encodes/encrypts a string using basic encoding (mode 1). If the third parameter is provided, it is used as a wizard parameter key to look up the data to encode.

DECODE — Decode String

Syntax: @TOOLS(DECODE, data) or @TOOLS(DECODE, data, output_key)

Decodes/decrypts a string using basic decoding (mode 1). If the third parameter is provided, stores the decoded result into that wizard parameter.

ENCODE_PARAM — Encode Parameter In-Place

Syntax: @TOOLS(ENCODE_PARAM, param_name)

Encodes a wizard parameter value in-place using stronger encoding (mode 3). Reads the parameter, encodes it, and returns the encoded value.

DECODE_DATA — Decode with Strong Mode

Executes a SQL script file in a secure context. This is one of the most commonly used @TOOLS sub-commands in production .sqi files. Internally it:

Behavior:

  6. Releases mutex
