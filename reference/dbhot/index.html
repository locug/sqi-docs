@dbHot Dispatch Modes

The @dbHot() command is the primary database and system state access command. It dispatches to 82 named modes based on its first argument, providing access to datasets, INI files, in-memory state, device info, configuration, and more.

Syntax: @dbHot(MODE, arg1 [, arg2 [, arg3 [, arg4]]])

Note

All @dbHot modes return empty strings or mock data in the browser REPL. This page documents what each mode does in the production POS environment.

Dataset Access (modes 11-16)

These modes read fields from the currently active dataset records. They all use the same generic field reader with the appropriate dataset pointer.

Syntax: @dbHot(DATASET, field [, format])

CodeModeDatasetDescription
11SAL_REGSale registerLine items in current sale
12SAL_HDRSale headerCurrent sale header record
13REC_REGReceiving registerReceiving line items
14REC_HDRReceiving headerCurrent receiving header
15INV_REGInventory registerInventory line items
16INV_HDRInventory headerCurrent inventory header

Format specifiers:

FormatMeaningExample
#Numeric (integer)@dbHot(SAL_HDR,F1032,#)
#:22 decimal places@dbHot(SAL_REG,F65,#:2)
!0:4Zero-padded to width 4@dbHot(SAL_HDR,F1032,!0:6)

Table Record Fallback

If the mode name is not in the dispatch table but matches a registered table name, @dbHot reads from that table directly. This is how @dbHot(POS_TAB,field) works.

The sub-command EXTSET writes to the table instead of reading:

Hot Wizard (73: HOT_WIZ)

Parameter passing between scripts. Uses a separate store from the main WIZGET/WIZSET store.

Syntax: @dbHot(HOT_WIZ, sub_command, key [, value])

Sub-commandSyntaxBehavior
GET@dbHot(HOT_WIZ,GET,key)Read value by key
CLR@dbHot(HOT_WIZ,CLR,key)Remove key
ISBLANK@dbHot(HOT_WIZ,ISBLANK,key)”1” if empty/missing, “0” if has value
RPLS@dbHot(HOT_WIZ,RPLS,key,value)String replace (mode 0)
RPLR@dbHot(HOT_WIZ,RPLR,key,value)Raw replace (mode 1)
RPLF@dbHot(HOT_WIZ,RPLF,key,value)Float replace (mode 2)
RPLT@dbHot(HOT_WIZ,RPLT,key,value)Mode variant
PARAMTOLINE@dbHot(HOT_WIZ,PARAMTOLINE,prefix)Serialize all params to string
LINETOPARAM@dbHot(HOT_WIZ,LINETOPARAM,key)Deserialize string back to params
RESET@dbHot(HOT_WIZ,RESET[,prefix])Reset matching entries

File Operations (42: FINDFIRST)

Check if files matching a pattern exist.

Syntax: @dbHot(FINDFIRST, pattern [, index])

Returns filename if found, empty if not. Optional index advances through results.

Register Operations (6: REG, 46-48: VAR*, 57: REGGEN, 64: REGSET)

REG — Read/Write Register Fields

Format specs: $#: for string, FLT for float, default for raw.

VARSET / VARGET / VARRESET

Use # prefix for array mode.

REGGEN — Sequence Numbers

Syntax: @dbHot(REGGEN, counter_name, params)

Reads/increments Windows registry counter at HKLM\SOFTWARE\Store Management\Counters\. Returns padded 8-digit sequence number.

REGSET — Create Register Field

Syntax: @dbHot(REGSET, fieldname, type, size) — returns “ERROR” on failure

Sub-commands: WIZGET, VALUE, PARAMS.
