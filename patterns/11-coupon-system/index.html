Pattern: Programmatic Coupons & Offers

What It Does

The coupon/offer system lets you programmatically create, match, apply, and destroy coupon discounts during a transaction. Instead of relying solely on scanned paper coupons, you can set up coupon parameters in the parameter store and trigger matching from SQI scripts.

This is the mechanism behind electronic coupons, loyalty offers, automatic discounts, and custom coupon programs.

Architecture

The coupon system has three layers:

  1. Parameter Setup — Set CPN_* variables in the parameter store to define what to match and how to discount
  2. @TOOLS Commands — Trigger coupon operations: COUPON_MATCH, COUPON_APPLY, COUPON_DESTROY, AUTO_START
  3. @dbHot(OFFER,…) — Query the current offer state: counts, status, quantities

The @TOOLS Coupon Commands

The core command. Set up CPN_* parameters first, then trigger the match:

Parameters read from the store:

ParameterRequiredDescription
CPN_FUNCTIONYesCoupon program number (e.g., 450). If not set, uses the default from the transaction
CPN_ELECTRONICNoSet to 1 for electronic coupons
CPN_MATCHCODEYesPrimary UPC/item code to match (20 chars max)
CPN_MATCHCODE1NoSecondary match code
CPN_MATCHCODE2NoTertiary match code
CPN_MATCHTYPENoHow to match: by item code, department, family, etc.
CPN_FAMILYNoPrimary family/group code

Discount parameters:

ParameterDescription
CPN_VALUEFixed discount amount (in cents)
CPN_PERCENTPercentage discount
CPN_FREESet to 1 for a free item coupon
CPN_BUNDLEBundle quantity (buy N get discount)

Constraint parameters:

CPN_MINAMT / CPN_MINAMT1 / CPN_MINAMT2Minimum amount required per match level
CPN_LIMTQTYMaximum quantity eligible for discount
CPN_LIMITDSCMaximum discount amount

CPN_UNLIMITEDSet to 1 for unlimited use

Exclusion parameters:

CPN_EXCLCODEComma or semicolon-separated exclusion item codes

COUPON_APPLY — Run Full E-Coupon Evaluation

Triggers the complete electronic coupon evaluation cycle on the current transaction:

What it does:

  1. Checks that the transaction is subtotaled (step 2)
  2. Saves the current parameter store state (backup)
  3. Looks for custom scripts matching trs_pos_ecpn_apply_*.sqi
  4. If a custom script exists, runs it instead of the built-in logic
  5. Otherwise, runs the built-in coupon evaluation
  6. Restores the parameter store backup

Tip

You can completely override the coupon apply logic by creating a script named trs_pos_ecpn_apply_custom.sqi (or any name matching the trs_pos_ecpn_apply_*.sqi pattern) in the script directory.

COUPON_DESTROY — Remove Applied Coupons

Reverses all coupon discounts from the current transaction:

What it does:

  1. Iterates through all line items with coupon flags
  2. Recalculates pricing without coupon discounts
  3. Resets all coupon accumulators and counters
  4. Looks for custom scripts matching trs_pos_ecpn_destroy_*.sqi

AUTO_START — Automatic Coupon Scan

Initiates an automatic coupon evaluation that scans all items in the transaction against all active offer rules:

Preconditions:

  • Transaction must be at step 2 (subtotaled)
  • Can be overridden with @WIZSET(ALLOWAUTOSTART=1)

Three-Level Matching

The coupon match engine supports three independent match levels, each with its own code, type, family, and quantity/amount constraints:

LevelMatch CodeMatch TypeFamilyMin QtyMin Amount
PrimaryCPN_MATCHCODECPN_MATCHTYPECPN_FAMILYCPN_MINQTYCPN_MINAMT
SecondaryCPN_MATCHCODE1CPN_MATCHTYPE1CPN_FAMILY1CPN_MINQTY1CPN_MINAMT1
TertiaryCPN_MATCHCODE2CPN_MATCHTYPE2CPN_FAMILY2CPN_MINQTY2CPN_MINAMT2

All three levels must match for the coupon to apply. Use this for complex rules like “Buy 2 of item A and 1 of item B, get $1 off.”

These parameter store flags control coupon behavior:

FlagEffect

Gotchas

  • Transaction must be subtotaled (step 2) before COUPON_APPLY or AUTO_START will work. Otherwise you get a TrsErrNoSubTotal / TrsErrModeRequired error.
  • COUPON_APPLY backs up and restores the parameter store — any @WIZSET calls inside trs_pos_ecpn_apply_*.sqi are temporary.
  • CPN_FUNCTION ranges matter — 450-459 and 460-469 use different receipt script sets. Other values use a default set.
  • CPN_MATCHCODE is 20 characters max — longer codes are truncated.
  • CPN_VALUE is in cents — set CPN_VALUE=100 for a $1.00 discount, not 1.00.
  • The match engine is re-entrant safe — it saves/restores offer state, so nested coupon operations work.
